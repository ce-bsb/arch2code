# arch2code webapp — runtime dependencies.
#
# Every package below is already installed on the interpreter this app targets
# (ARCH2CODE_PYTHON, default /opt/anaconda3/bin/python). The pins are loose
# lower bounds so an offline machine that already satisfies them needs no
# network at all: `./run.sh` must work with the network cable pulled.
#
# Verified present on this machine:
#   fastapi 0.129.0  uvicorn 0.40.0  python-multipart 0.0.22  pydantic 2.12.4
#   mcp 1.26.0       httpx 0.28.1    pillow 12.0.0            starlette 0.52.1

fastapi>=0.115
uvicorn[standard]>=0.30
python-multipart>=0.0.9
pydantic>=2.7

# MCP stdio client for mcp/arch_vision/server.py (health handshake + Mode A).
mcp>=1.2
httpx>=0.27

# Image probing for the bbox overlay base layer. capture_diagram.py needs it too,
# in the interpreter named by ARCH2CODE_PYTHON.
pillow>=10.0

# ---------------------------------------------------------------------------
# Universal format ingestion (app/ingest).
#
# Every one of these is a pure-Python or self-contained wheel. NO apt-get, NO
# brew, NO system binary — that is a hard constraint, not a preference: the app
# has to come up with one command on a laptop and inside a minimal container.
# Deliberately NOT installed, and why:
#   python-magic   needs libmagic1 for a job `filetype` does in pure Python
#   pdf2image      needs poppler-utils; pypdfium2 does the same in-process
#   PyMuPDF        AGPL-or-commercial, and this IS a network service (see
#                  app/ingest/adapters/pdf.py); pypdfium2 is Apache-2.0/BSD-3
#   graphviz, JRE  only needed to *render* PlantUML, which we read as text
#   LibreOffice    ~400 MB, needs font packages or it renders empty boxes, and
#                  is not concurrency-safe; the formats needing it are refused
#                  with conversion instructions instead
# ---------------------------------------------------------------------------

# Magic-byte type detection. Pure Python, zero dependencies, reads 261 bytes.
filetype>=1.2

# HEIC/HEIF decoding — every iPhone photo, and the single most likely real input
# for a whiteboard. Ships manylinux/macOS wheels with libheif bundled.
# Without this, .heic uploads raise UnidentifiedImageError: the routing tables,
# capture_diagram.py and the upload endpoint all advertised .heic before it was
# a dependency.
pillow-heif>=0.16

# PDF text, geometry and rasterization, in-process. Apache-2.0/BSD-3.
pypdfium2>=4.0

# Visio 2013+ (.vsdx): real shapes and real connectors. Pure Python (Jinja2 only).
vsdx>=0.5

# Tests only (webapp/tests). Not required to run the app.
pytest>=8.0
